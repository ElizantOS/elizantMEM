# Memory Policy Workspace

This is the editable surface for the autonomous research loop.

Artifacts:

- `literature/review.md`: prior work summary and research gap extraction
- `literature/benchmark-map.md`: benchmark-to-metric mapping from papers to this project
- `paper/abstract.md`: concise paper pitch
- `paper/outline.md`: full narrative structure
- `design/memory-loop.md`: system architecture and policy details
- `experiments/plan.md`: experiment strategy and rollout phases
- `experiments/eval-matrix.md`: baselines, metrics, ablations, and outcome table
- `results/readout.md`: running result table and next hypotheses

The controller accepts only files inside this directory.

Expected flow:

1. Ground the project in literature and public benchmarks.
2. Derive the metric contract and experiment matrix from those sources.
3. Tighten architecture and paper artifacts without drifting away from cited evidence.
