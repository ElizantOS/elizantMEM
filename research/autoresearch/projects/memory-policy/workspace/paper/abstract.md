# Abstract Draft

We propose a long-term memory framework for task-oriented agents in which memory is optimized as a decision system rather than treated as passive storage. The target setting is a data-analysis assistant that operates across repeated tasks, changing schemas, user preferences, and recurring error modes. In that setting, retaining more information is not sufficient: the agent must decide what evidence is worth writing into memory, what existing memory is worth retrieving for the current task, and when previously useful memory has become stale, redundant, or harmful. Our core claim is that long-term memory should be modeled as two coupled policies, write policy and retrieval policy, connected by a lifecycle feedback ledger that measures downstream utility.

The proposed architecture contains eight modules: event capture, candidate generation, memory scoring, write policy, typed memory store, retrieval policy, execution contextualization, and feedback ledger. The write policy operates over candidate memories extracted from successful traces, user corrections, schema discoveries, and repeated workflows. The retrieval policy decides whether memory should be injected at all, which memory types are admissible for the current task, and how much context budget should be spent. The feedback ledger records whether a written or retrieved memory later improved task success, SQL correctness, retry count, token cost, or latency, and whether it caused pollution, duplication, or stale-context failures. This turns memory into a measurable control loop with explicit rewards, constraints, and failure modes.

We outline a staged learning path from rule-based policies to contextual bandits and then multi-step reinforcement learning. We also define an experiment program that separates write quality from retrieval quality, includes offline replay plus online evaluation, and compares passive vector memory against typed rule systems and learned policy variants. The resulting research agenda aims to produce a paper-ready argument, an implementation-ready system design, and a falsifiable benchmark suite for memory as policy.
