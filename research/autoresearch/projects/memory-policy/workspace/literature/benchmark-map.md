# Benchmark Map

| Benchmark / Paper | Task | Metrics or abilities | What it measures | Relevance to memory-policy | Notes |
| --- | --- | --- | --- | --- | --- |
| MemGPT (2023) | Stateful agent memory management | No single standard benchmark; architecture baseline | Hierarchical memory organization and paging discipline | Baseline for layered memory architecture | Use as heuristic baseline rather than RL baseline |
| LoCoMo (arXiv:2402.17753, 2024-02-27) | Very long-term conversational memory | Long-horizon dialogue consistency and temporal understanding | Whether the agent preserves long-range conversational facts | Useful for testing temporal consistency and persona persistence | Good source for memory drift scenarios |
| LongMemEval (arXiv:2410.10813, 2024-10-14) | Long-term interactive chat assistant memory | Information extraction, multi-session reasoning, temporal reasoning, knowledge updates, abstention | Whether memory helps over extended interaction windows | Strong benchmark source for retrieval and abstention metrics | Its ability split can map into our eval dimensions |
| Memory-R1 (arXiv:2508.19828, 2025-08-27) | RL-managed memory agents | Downstream task performance and generalization | Whether learned memory actions beat static heuristics | Closest baseline for learned memory manager comparisons | Track learned operations against heuristic store/retrieve baselines |
| Mem-α (arXiv:2509.25911, 2025-09-30) | RL memory construction | Memory construction quality plus downstream task results | Whether the agent structures memory effectively | Relevant for write policy and memory-unit construction | Suggests evaluating structure quality, not only retrieval success |
| AgeMem (arXiv:2601.01885, 2026-01-05) | Unified short-/long-term memory policy | Policy-level memory-operation quality | Whether memory operations are learned jointly inside the agent policy | Closest conceptual anchor for "Memory is Policy" | Helps justify unified policy framing |
| MemRL (arXiv:2601.03192, 2026-01-06) | Runtime RL on episodic memory | Utility-aware retrieval and task success | Whether utility-aware retrieval beats passive semantic retrieval | Strong reference for feedback ledger and retrieval utility | Distinguish retrieval relevance from retrieval value |
| AtomMem (arXiv:2601.08323, 2026-01-13) | Atomic memory CRUD control | Decision quality over create/read/update/delete operations | Whether decomposed memory actions improve flexibility | Useful for explicit write/retrieve/update/delete action spaces | Good source for action-space justification |
