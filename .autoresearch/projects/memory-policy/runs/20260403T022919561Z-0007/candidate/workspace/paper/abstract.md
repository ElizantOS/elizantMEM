# Abstract Draft

We propose a long-term memory system for task-oriented agents where memory is optimized as a policy rather than a passive store. In data-analysis workflows, useful memory is determined not by raw retention but by whether candidate memories are worth writing, worth retrieving, and worth preserving under future task utility. We decompose the system into event capture, candidate generation, memory scoring, write policy, typed memory storage, retrieval policy, execution-time contextualization, and a feedback ledger that supports credit assignment over future tasks. This framing turns agent memory into a measurable closed loop with explicit tradeoffs between success lift, efficiency, and memory pollution. We outline an implementation path from rule-based heuristics to contextual bandits and finally multi-step reinforcement learning, and define an evaluation suite over task success, SQL success, token cost, latency, retrieval utility, and stale-memory risk.
