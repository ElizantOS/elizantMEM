# Abstract Draft

We propose a long-term memory architecture for task-oriented agents in which memory is optimized as a decision system rather than a passive store. In a data-analysis assistant, the central problem is not how to retain more traces, but how to decide which observations deserve durable storage, which memories should be surfaced for a new task, and when an existing memory should be merged, decayed, or ignored. The key claim is that long-term memory must be modeled as two coupled policies, write policy and retrieval policy, with lifecycle feedback that measures whether prior memory decisions improved future task outcomes.

Our design decomposes the loop into eight modules: event capture, candidate generation, memory scoring, write policy, memory store, retrieval policy, execution contextualization, and a feedback ledger with explicit reward accounting. The write policy chooses among skip, write, merge, replace, and time-to-live actions over typed memory units such as schema knowledge, user preferences, reusable query patterns, and exception handling strategies. The retrieval policy decides whether to retrieve at all, which memory types to consult, and how much context budget to allocate. The feedback ledger links those decisions to delayed outcomes including task success, SQL success, retry count, token cost, latency, stale retrievals, and memory pollution.

This framing treats memory as a product surface that changes agent behavior, not as background infrastructure. It also makes memory falsifiable: a write is good only if it improves future decisions enough to justify storage and retrieval cost, and a retrieval is good only if its inclusion improves execution more than it distracts or bloats context. We outline a learning path from rule-based policies, to contextual bandits for local decisions, to multi-step reinforcement learning over the full memory loop. We also define an experiment program with offline replay, online task evaluation, explicit baselines, and policy ablations so that gains in write precision, retrieval utility, task success lift, token reduction, and latency reduction can be measured directly rather than inferred from anecdotal agent behavior.
